# B-Hall UI/UX設計原則

## 目指す質感

Mac / iOSアプリのような静かな高級感。
ガラスコーティング感。半透明。ブラー。柔らかな境界線。ミニマル。
情報整理が上手い。触りたくなる業務システム。

## 設計原則

### 1. 1画面1目的
各画面は一つの目的を持つ。複数の目的を詰め込まない。

### 2. 次にやることが分かる
ダッシュボードの最上部は常に「今すぐ対応すべきこと」。

### 3. 情報の階層化
- Level 1: 一覧・サマリー（スキャンで把握）
- Level 2: カード・セクション（注目して読む）
- Level 3: 詳細画面（じっくり操作する）

### 4. コンテキストの維持
画面遷移時に「どこから来たか」を見失わない。パンくずリスト、戻るナビゲーション。

### 5. 空状態のデザイン
データがない状態も美しく、次のアクションを案内する。

### 6. 状態の可視化
- 承認待ち → 黄色系アクセント
- 差戻し → 赤系アクセント
- 完了 → 緑系アクセント
- 期限超過 → 赤系アラート

## 構造ルール

### サイドバー
- 常時表示（デスクトップ）
- 折りたたみ可能
- アイコン + テキスト
- アクティブ状態の明示
- 未読/要対応のバッジ表示

### カードUI
- カード間に十分な余白
- カード内は情報を厳選（3-5項目）
- ホバーで追加情報表示
- クリックで詳細遷移

### モーダル
- 確認ダイアログ、簡易編集のみ
- 詳細な操作は専用画面へ
- 背景のブラー効果

### テーブル
- 使用は最小限
- 使う場合もカード的なスタイリング
- ゼブラストライプではなく、ホバーハイライト
- ページネーションよりも無限スクロール

### レスポンシブ
- デスクトップ: サイドバー + メインコンテンツ
- タブレット: 折りたたみサイドバー + メインコンテンツ
- スマートフォン: ボトムナビ + カード中心 + ドロワー

## 禁止事項

1. 安っぽいSaaSテンプレートの流用
2. 情報を詰め込みすぎたダッシュボード
3. テーブルだけで構成された画面
4. 同じサイズのカードを均一に並べただけの画面
5. どこかで見た管理画面の焼き直し
6. 装飾のないプレーンなフォーム
7. コントラスト不足で読みにくいテキスト
8. 意味のないアニメーション

## インタラクション

- ボタン: ホバー時に微細なスケール変化
- カード: ホバー時に影の変化
- 画面遷移: スムーズなフェードトランジション
- ローディング: スケルトンスクリーン（スピナーではない）
- トースト通知: 右上に表示、3秒で自動消去
