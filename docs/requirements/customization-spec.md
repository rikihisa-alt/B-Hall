# B-Hall カスタマイズ仕様

## 概要

中小企業は会社ごとに運用が異なる。B-Hallは固定機能だけでなく、企業ごとの運用に合わせたカスタマイズを提供する。

## カスタム項目

### 対象エンティティ
- タスク
- 申請
- 稟議
- 従業員
- 取引
- 文書
- 報告

### フィールド型

| 型 | 用途 |
|----|------|
| text | 短いテキスト |
| textarea | 長いテキスト |
| number | 数値 |
| date | 日付 |
| select | 単一選択 |
| multi_select | 複数選択 |
| checkbox | チェックボックス |
| user | ユーザー参照 |
| file | ファイル添付 |

### 管理方式
- テナント管理者がUI上でカスタム項目を定義
- JSON型カラム（custom_fields）に格納
- カスタム項目でのフィルタ・ソート対応

## カスタムカテゴリ

- タスクの業務カテゴリ・サブカテゴリをテナント単位で定義
- デフォルトカテゴリを提供しつつ、追加・編集・削除可能

## カスタムタスクテンプレート

- テナント単位でタスクテンプレートを作成
- テンプレートには複数タスク + チェックリストを含む
- イベントトリガーとの紐付け
- 会社別テンプレート: 同一テナント内の会社（子会社等）ごとに異なるテンプレート

## カスタム申請フォーム

- フォームビルダーでUIから定義
- カスタム項目の組み合わせでフォームを構成
- 承認ルートとの紐付け
- テンプレートとしての保存・再利用

## カスタム承認ルート

- 申請種別ごとに承認ルートを定義
- 条件分岐（金額、部署等）
- 承認者の指定（ユーザー、ロール、役職）

## 部署別表示切替

- サイドバーの表示モジュールを部署ごとにカスタマイズ
- 経理部門はM08を最上位表示、人事部門はM05を最上位表示

## ロール別表示調整

- ダッシュボードのウィジェット構成をロールごとにカスタマイズ
- 不要なメニュー項目の非表示設定
