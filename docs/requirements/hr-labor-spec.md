# B-Hall 人事・労務仕様

## 概要

従業員に関するすべての手続き・状態・リスクを漏れなく管理する。
イベント発生→タスク自動生成→書類チェック→進捗管理→行政期限アラートの連動が核心。

## 従業員マスタ

→ `domain-model.md` のEmployee定義を参照

### 追加管理項目
- 保有資格一覧
- 健康診断履歴
- ストレスチェック履歴
- 有休残日数・付与履歴
- 休職・復職履歴
- 評価履歴
- 異動履歴
- 契約更新履歴

## 人事イベント一覧

| イベント | 自動生成タスク | 必要書類 | 行政期限 |
|---------|-------------|---------|---------|
| 入社 | 雇用契約書回収、社保手続き、雇保手続き、備品手配 | 雇用契約書、マイナンバー、年金手帳写し | 社保：5日以内 |
| 退社 | 社保喪失届、雇保喪失届、離職票作成、備品返却 | 退職届、社保喪失届 | 喪失届：5日以内 |
| 異動 | 辞令作成、座席変更、組織図更新 | 辞令書 | - |
| 昇格 | 辞令作成、給与改定 | 辞令書 | - |
| 契約更新 | 契約書作成、条件確認 | 雇用契約書 | 更新1ヶ月前通知 |
| 社保加入 | 資格取得届提出 | 資格取得届 | 5日以内 |
| 年末調整 | 扶養控除申告書回収、保険料控除申告書回収 | 各種申告書 | 翌年1月末 |
| 健康診断 | 受診手配、結果回収 | 健診結果 | 年1回 |
| 36協定 | 労使協定書作成、届出 | 36協定書 | 毎年届出 |

## 連動フロー例：入社

```
入社イベント発生
  ├── タスク: 雇用契約書の締結（人事担当、3日以内）
  ├── タスク: 社保加入手続き（労務担当、5日以内）
  ├── タスク: 雇保加入手続き（労務担当、翌月10日まで）
  ├── タスク: 備品・端末手配（総務担当、入社日まで）
  ├── タスク: アカウント発行（IT担当、入社日まで）
  ├── タスク: 入社オリエン実施（人事担当、入社日）
  └── 通知: 関係者への入社通知
```

## 有休管理

- 法定付与日数の自動計算（入社日起算）
- 取得実績の記録
- 残日数のリアルタイム表示
- 時効消滅の管理（2年）
- 年5日取得義務の監視・アラート
