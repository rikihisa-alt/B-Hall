# B-Hall 報告・改善仕様

## 概要

問題発見と改善活動を日常業務に組み込む。報告は投稿で終わらず、改善タスク化→再発防止→経営可視化まで繋がる。

## 報告種別

| 種別 | 用途 | 匿名可 | 自動タスク化 |
|------|------|--------|------------|
| 日報 | 日次業務報告 | × | × |
| 週報 | 週次業務報告 | × | × |
| 月報 | 月次業務報告 | × | × |
| 事故報告 | 業務上の事故 | × | ○ |
| インシデント報告 | セキュリティ・情報事故 | × | ○ |
| クレーム報告 | 顧客クレーム | × | ○ |
| ヒヤリハット | 事故未遂の報告 | ○ | ○ |
| 改善提案 | 業務改善の提案 | ○ | ○ |
| 目安箱 | 自由な意見投稿 | ○ | △ |

## 報告→改善フロー

```
報告投稿
  → 対応者の自動/手動割当
  → 対応状況の追跡
  → 改善タスク作成（M02と連携）
  → 再発防止タスク作成
  → 経営者への通知・可視化（M01/M10）
  → 改善テーマとして蓄積（Layer 4）
```

## 匿名投稿

- ヒヤリハット、改善提案、目安箱で匿名投稿を許可
- 匿名投稿では投稿者のIDを記録しない（復元不可）
- 管理者でも投稿者を特定できない設計とする
- ただし、対応コメントは実名で行う

## 改善テーマ管理

報告から生まれた改善テーマを集約・分類する。
- テーマごとの改善進捗
- テーマごとの報告件数推移
- 未対応テーマのアラート
- 経営ダッシュボード（M01/M10）への表示

## ビュー

- 報告一覧（種別・ステータスフィルタ）
- 報告詳細（対応タイムライン）
- 改善テーマボード
- 報告統計（件数推移、種別分布）
