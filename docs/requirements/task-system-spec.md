# B-Hall タスクシステム仕様

## 位置づけ

タスクはB-Hall全体のハブ。すべての業務オブジェクト（申請、稟議、書類、取引、報告）はタスクと紐付く。

## データモデル

→ `domain-model.md` のTask定義を参照

## ステータス遷移図

```
                  ┌──── 保留 ←───┐
                  ↓              │
未着手 → 進行中 → 確認待ち → 承認待ち → 完了
          ↑         │          │
          └── 差戻し ←┘←────────┘
                  ↓
                中止
```

## ビュー

### 一覧ビュー
- フィルタ: ステータス、担当者、カテゴリ、期限、優先度
- ソート: 期限順、優先度順、更新日順
- 一括操作: ステータス変更、担当者変更

### ボードビュー（カンバン）
- カラム: ステータス別
- ドラッグ&ドロップでステータス変更
- カードにはタイトル、担当者、期限、優先度を表示

### 期限ビュー
- タイムライン形式で期限を可視化
- 期限超過を赤、期限接近を黄色で表示

### カードビュー
- 詳細情報をカード形式で表示
- 関連オブジェクト（申請、稟議、書類）へのリンク

## テンプレート

```typescript
interface TaskTemplate {
  id: string
  name: string
  category: string
  tasks: TaskDefinition[]  // 複数タスクをまとめて定義可能
}

interface TaskDefinition {
  title: string
  description?: string
  checklist?: ChecklistItem[]
  assignee_rule: 'creator' | 'role' | 'specific_user'
  assignee_value?: string
  due_offset_days?: number  // イベント日からの日数
  priority: string
  sub_tasks?: TaskDefinition[]
}
```

## 自動生成トリガー

| イベント | 生成タスク例 |
|---------|------------|
| 入社 | 雇用契約書回収、社保手続き、備品手配、アカウント発行、入社オリエン |
| 退社 | 退職届受領、社保喪失届、備品返却、アカウント停止、最終給与計算 |
| 月末 | 経費締め、勤怠確認、請求書発行、支払実行確認 |
| 稟議承認 | 契約書手配、支払依頼作成、実行計画策定 |
| 契約更新30日前 | 更新判断、条件交渉、契約書準備 |

## チェックリスト

各タスクに複数のチェックリスト項目を持てる。
チェックリストの全項目完了が、タスク完了の前提条件として設定可能。
