# B-Hall 情報アーキテクチャ

## システム全体構造

B-Hallは5つのレイヤーで構成される。

```
┌─────────────────────────────────────────────┐
│  A. 経営レイヤー                              │
│  M01 ダッシュボード / M10 経営管理             │
├─────────────────────────────────────────────┤
│  B. 業務統制レイヤー                           │
│  M02 タスク / M03 ワークフロー / M04 稟議       │
│  M09 報告・改善 / M12 通知・期限               │
├─────────────────────────────────────────────┤
│  C. 部門業務レイヤー                           │
│  M05 人事労務 / M06 総務 / M07 法務文書         │
│  M08 経理財務                                 │
├─────────────────────────────────────────────┤
│  D. ナレッジ・標準化レイヤー                    │
│  M11 ナレッジ・テンプレート                     │
├─────────────────────────────────────────────┤
│  E. 基盤レイヤー                              │
│  M13 組織・権限・監査                          │
│  認証 / 通知基盤 / ファイル基盤 / WFエンジン     │
│  カスタム項目基盤 / ログ基盤                     │
└─────────────────────────────────────────────┘
```

## モジュール一覧

| ID | モジュール名 | レイヤー | 概要 |
|----|------------|---------|------|
| M01 | ダッシュボード | A | ロール別ホーム画面 |
| M02 | タスク管理 | B | システム全体のハブ |
| M03 | ワークフロー | B | 申請・承認の標準化 |
| M04 | 稟議 | B | 意思決定の記録・統制 |
| M05 | 人事・労務 | C | 人に関する手続き・状態管理 |
| M06 | 総務・庶務 | C | 日常運営管理の標準化 |
| M07 | 法務・契約・文書 | C | ビジネス文書の一元管理 |
| M08 | 経理・財務 | C | 経理実務と資金管理 |
| M09 | 報告・改善 | B | 問題発見と改善活動 |
| M10 | 経営管理 | A | 経営判断支援 |
| M11 | ナレッジ・テンプレート | D | 業務標準化 |
| M12 | 通知・期限管理 | B | 通知配信と期限監視 |
| M13 | 組織・権限・監査 | E | 基盤管理 |

## モジュール依存関係

### 基盤依存（すべてのモジュールが依存）
- M13 組織・権限・監査
- 認証基盤
- 通知基盤（M12）
- ファイル基盤

### ハブ依存（多くのモジュールが依存）
- M02 タスク管理：すべてのモジュールからタスクが生成される

### 上位依存（集約して表示するモジュール）
- M01 ダッシュボード：全モジュールの情報を集約
- M10 経営管理：M02, M04, M05, M08の情報を集約

## 画面遷移の基本原則

1. ダッシュボードから各モジュールへ1クリックで遷移
2. 関連オブジェクト間はインラインリンクで遷移
3. モーダルは確認・簡易編集のみ。詳細は専用画面へ
4. ブラウザの戻る/進むを正しくサポート
5. URLはすべてのページに対して一意（ディープリンク対応）
