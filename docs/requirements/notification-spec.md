# B-Hall 通知・期限管理仕様

## 概要

必要な情報を必要な人に必要なタイミングで届ける。通知が多すぎてノイズになることを防ぐ設計。

## 通知チャネル

| チャネル | 用途 | 実装時期 |
|---------|------|---------|
| アプリ内通知 | リアルタイム通知 | P0 |
| メール | 重要通知・ダイジェスト | P1 |
| Webhook | 外部連携 | P2 |
| Slack連携 | チャット通知 | P2 |

## 通知種別と優先度

| 種別 | 優先度 | アプリ内 | メール |
|------|--------|---------|--------|
| タスク割当 | 中 | ○ | △ |
| 期限接近（3日前） | 高 | ○ | ○ |
| 期限超過 | 緊急 | ○ | ○ |
| 承認依頼 | 高 | ○ | ○ |
| 差戻し | 高 | ○ | ○ |
| コメント | 低 | ○ | △ |
| 書類未提出 | 中 | ○ | △ |
| 契約更新期限 | 高 | ○ | ○ |
| 行政手続き期限 | 緊急 | ○ | ○ |
| 支払期限 | 高 | ○ | ○ |
| 入金遅延 | 高 | ○ | ○ |
| キャッシュ不足予兆 | 緊急 | ○ | ○ |
| 重要報告投稿 | 高 | ○ | ○ |

△ = ユーザー設定による

## 重複制御

### グルーピング
- 同一ソースからの連続通知をグルーピング
- 例：同一タスクへの複数コメント → 「3件の新しいコメント」

### ダイジェスト
- 低優先度通知は日次ダイジェストメールにまとめる
- 朝9時にダイジェスト配信

### ミュート
- 通知種別ごとにミュート可能
- 特定タスク/プロジェクトのミュート

## 期限管理

### 期限監視バッチ
- 日次バッチで全タスク・手続きの期限をチェック
- 期限3日前、1日前、当日、超過のタイミングで通知

### 行政手続き期限
- 社保届（5日以内）、雇保届（翌月10日）等の法定期限を管理
- 期限切れリスクの高い手続きを優先表示
