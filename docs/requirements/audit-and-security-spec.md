# B-Hall 監査・セキュリティ仕様

## 認証

### 認証方式
- メールアドレス + パスワード（初期）
- 将来: SSO（SAML 2.0）、OAuth 2.0対応

### セッション管理
- JWTベースのトークン認証
- アクセストークン有効期限: 15分
- リフレッシュトークン有効期限: 7日
- 同時セッション数制限: テナント設定可能

### パスワードポリシー
- 最小8文字
- 英大文字・小文字・数字を含む
- 過去5世代のパスワード再利用禁止

## 認可

### RBAC（ロールベースアクセス制御）
→ `roles-and-permissions.md` を参照

### テナント分離
→ `multitenancy-spec.md` を参照

### データアクセス制御
- すべてのAPIリクエストでテナントID検証
- ロールに基づく操作権限チェック
- 部門スコープでのデータフィルタリング

## 監査ログ

### 記録対象

| 操作 | ログ内容 |
|------|---------|
| 作成 | 対象エンティティ、作成者、タイムスタンプ |
| 編集 | 対象エンティティ、変更前後の値、編集者 |
| 削除 | 対象エンティティ、削除者 |
| 承認 | 申請/稟議ID、承認者、承認結果 |
| 差戻し | 申請/稟議ID、差戻し者、理由 |
| アップロード | ファイル名、サイズ、アップロード者 |
| 権限変更 | 対象ユーザー、変更前後の権限 |
| ログイン | ユーザー、IPアドレス、ユーザーエージェント |
| エクスポート | エクスポート種別、対象データ範囲、実行者 |

### ログ保持期間
- 最低90日（デフォルト）
- プレミアムプラン: 365日

### ログの閲覧権限
- CEO, ADMIN, AUDITロールのみ

## データ保護

- 通信の暗号化: TLS 1.3
- データの暗号化: AES-256（保存時）
- 個人情報のマスキング: ログ内の個人情報は部分マスク
- バックアップの暗号化

## 外部専門家アクセス

- AUDIT ロールで限定的なアクセスを提供
- アクセス可能範囲: 監査ログ、文書（権限付与されたもの）、証跡
- アクセス期間を設定可能（開始日〜終了日）
- すべてのアクセスをログに記録
