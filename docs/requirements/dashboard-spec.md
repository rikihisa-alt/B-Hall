# B-Hall ダッシュボード仕様

## 概要

M01ダッシュボードはロール別の「ホーム画面」。情報を詰め込まず、各ロールに必要な情報だけを表示する。
M10経営管理はドリルダウン先の「経営分析・判断画面」。

## M01 ダッシュボード

### 経営者ビュー

**構成（上から順に）**:

1. **アラートバー**: 緊急対応が必要な項目（期限超過、キャッシュ不足予兆等）
2. **財務サマリー**: 現預金残高、月次収支、資金繰り予測（簡易グラフ）
3. **要対応セクション**: 未承認稟議、高額支出予定、未処理法定手続き
4. **人員セクション**: 従業員数推移、離職/休職動向
5. **リスク・改善セクション**: リスク報告件数、改善提案件数、重要インシデント
6. **部門別滞留**: 各部門の未完了タスク・承認待ち件数

各セクションからM10経営管理の詳細画面へドリルダウン可能。

### 管理者ビュー

1. **アクション必要**: 承認待ち一覧、期限超過一覧
2. **進捗状況**: 部門別タスク進捗、手続き完了率
3. **未提出**: 日報/報告未提出者、コメント未対応
4. **月次**: 月次締め状況チェックリスト

### 担当者ビュー

1. **今日やること**: 期限が今日のタスク、承認待ちの自分の申請
2. **マイタスク**: 自分が担当のタスク一覧（ステータス別）
3. **通知**: 差戻し、未読コメント
4. **期限接近**: 今週中に期限が来るもの

### Backllyビュー

1. **顧客ヘルス**: 顧客別ヘルススコア（利用率、滞留率、未処理率から算出）
2. **要介入**: 詰まり企業一覧、要介入タスク
3. **改善提案**: 次に提案すべき改善テーマ

## M10 経営管理

M01からのドリルダウン先。以下をタブ構成で提供:

- **財務**: 売上/支出サマリー、資金繰り詳細、予算対比
- **人事**: 人員増減詳細、離職分析、組織図
- **業務**: 部門別業務負荷、タスク完了率、承認リードタイム
- **リスク**: リスク一覧、改善テーマ進捗、インシデント分析
- **投資**: 稟議とキャッシュの紐付け、投資後の実行進捗、効果検証

## 設計原則

- 1画面に情報を詰め込まない
- スクロールは許容するが、最重要情報は画面上部に
- 数値はグラフ/チャートで直感的に
- 異常値は色・アイコンで即座に認識可能に
- すべてのデータはドリルダウン可能
