# B-Hall 申請・承認・稟議仕様

## 申請・承認（M03）

### 対応申請種別

| 種別 | 概要 | 承認後の連携 |
|------|------|------------|
| 経費申請 | 経費精算の申請 | →M08取引作成 |
| 交通費申請 | 交通費の申請 | →M08取引作成 |
| 休暇申請 | 有休・特休等 | →M05有休管理更新 |
| 残業申請 | 残業の事前申請 | →M05労務記録 |
| 出張申請 | 出張の事前申請 | →M08経費予定 |
| 購買申請 | 備品・物品購入 | →M06備品管理、M08支払 |
| 契約確認 | 契約内容の確認申請 | →M07契約管理 |
| 支払依頼 | 支払の実行依頼 | →M08支払管理 |
| 採用申請 | 採用の承認依頼 | →M05入社フロー |
| 端末・アカウント発行 | IT関連の発行依頼 | →M06総務タスク |
| 稟議起票 | 稟議の申請 | →M04稟議フロー |
| 社内制度利用 | 福利厚生等 | →関連タスク |
| 自由申請 | カスタム申請 | →設定による |

### 申請フロー

```
起票（下書き保存可）
  → 申請提出
  → 承認ステップ1（承認/差戻し）
  → 承認ステップ2（条件付き）
  → ...
  → 最終承認
  → 完了処理（タスク自動生成、取引作成等）
```

### 差戻し・再申請

- 差戻し時にコメント必須
- 再申請時に変更箇所のハイライト
- 差戻し→再申請の履歴を保持

## 稟議（M04）

### 稟議フィールド

- 件名、背景、目的、内容
- 金額（概算/確定）
- 投資回収見込み
- リスク評価
- 期待効果
- 関係部門
- 添付資料
- 承認ルート

### 稟議承認後の必須連携チェーン

```
稟議承認
  ├── 契約書の紐付け/作成依頼（→M07）
  ├── 支払依頼の作成（→M08）
  ├── 実行タスクの生成（→M02）
  └── 効果検証メモの作成予約（→M02、3ヶ月後等）
```

この連携チェーンが切断されていないことをシステムが監視する。
未完了の連携がある場合、稟議起票者と経営者に通知。

### 稟議ビュー

- 稟議一覧（ステータス別フィルタ）
- 稟議詳細（承認経過のタイムライン表示）
- 金額別サマリー（経営者向け）
- 実行状況トラッキング（承認後の進捗）
