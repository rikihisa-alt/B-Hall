## 1. プロダクト概要

### 1-1. サービス名

B-Hall（ビーホール）

### 1-2. 提供元

株式会社Backlly

### 1-3. 定義

営業活動以外の中小企業バックオフィス全体を統合し、出来事起点で業務・承認・書類・お金・人・改善活動をつなぐクラウド型バックオフィスOS。

### 1-4. ターゲット

従業員5〜300名規模の中小企業。社労士・税理士等の外部専門家との協業も想定。

### 1-5. 核心的価値

機能の多さではなく、バックオフィスのすべてが一つの企業運営フローとして繋がっていること。

## 3. 5層アーキテクチャモデル

### Layer 1: イベント層

業務の発生起点。すべてのフローはここから始まる。

**イベント種別一覧**:

| カテゴリ | イベント |
|----------|---------|
| 人事 | 入社、退社、異動、昇格、休職、復職 |
| 労務 | 社保加入/喪失、雇保加入/喪失、労災、年末調整、36協定 |
| 契約 | 契約締結、契約更新、契約終了 |
| 経理 | 支払発生、請求発生、入金、月次締め、決算 |
| 経営 | 投資判断、設備購入、規程改定 |
| 総務 | 備品発注、端末貸与、アカウント発行 |
| 報告 | 事故発生、インシデント、クレーム、改善提案 |
| 定期 | 月初、月末、四半期、年度末 |

### Layer 2: フロー層

イベントに紐づく処理経路を定義。

**フロー定義要素**:
- トリガーイベント
- 起票者（自動/手動）
- 生成されるタスク群
- 必要書類
- 承認ルート
- 関与部門
- 期限ルール
- 完了条件

### Layer 3: オブジェクト層

フローの中を流れる業務オブジェクト。

**コアオブジェクト**: タスク、申請、承認、稟議、報告、書類、取引、コメント、通知

**オブジェクト間リレーション**: すべてのオブジェクトはタスクを起点に相互参照可能。

### Layer 4: 経営情報層

フロー実行結果として蓄積される情報。

進捗、人員状況、手続き状況、コスト、キャッシュ、リスク、未処理件数、滞留、改善テーマ

### Layer 5: 経営判断層

蓄積情報を元に経営者が判断・改善を行うレイヤー。

## 6. モジュール別要件

### M01 ダッシュボード

**目的**: ロール別に「今必要な情報」だけを表示するホーム画面。

**設計原則**: 情報を詰め込まない。ロール別に異なるビュー。M10経営管理へのエントリーポイント。

**各ロールの表示内容**: 本プロンプト内のM01定義に準拠。

**技術要件**: リアルタイム更新（WebSocket/SSE）、ウィジェット構成のカスタマイズ可。

---

### M02 タスク管理

**目的**: システム全体のハブ。すべての業務オブジェクトはタスクと紐付く。

**エンティティ**: Task

**フィールド**: 本プロンプト内のM02定義に完全準拠。

**ステータス遷移**:
```
未着手 → 進行中 → 確認待ち → 承認待ち → 完了
                              ↓              ↑
                           差戻し → 進行中 ─┘
         ↕
        保留
         ↓
        中止
```

**ビュー**: 一覧、ボード（カンバン）、期限ビュー（タイムライン）、カード

**自動生成**: イベントトリガー + テンプレートIDで自動生成。テンプレートはタスク群（複数タスク+チェックリスト）を定義可能。

---

### M03 ワークフロー

**目的**: 申請→承認の経路を標準化し、全バックオフィス申請をデジタル化。

**エンティティ**: Application, ApprovalRoute, ApprovalStep

**承認ルート設計**:
- 直列承認（A→B→C）
- 並列承認（A+B→C）
- 条件分岐（金額X以上→役員承認追加）
- 代理承認

**ステータス**: 下書き→申請中→承認中→承認済み→差戻し→再申請→取下げ

**連携**: 承認完了→関連タスク自動生成、関連取引自動作成

---

### M04 稟議

**目的**: 会社の意思決定を記録・可視化・統制。

**エンティティ**: Ringi

**必須連携チェーン**: 稟議承認→契約書紐付け→支払依頼作成→実行タスク生成→効果検証メモ

---

### M05 人事・労務

**目的**: 人に関するすべての手続き・状態・リスクを管理。

**エンティティ**: Employee, HREvent, LaborProcedure

**自動連動**: イベント発生→タスク自動生成→書類チェックリスト→提出進捗→行政期限アラート

---

### M06 総務・庶務

**目的**: 日常の会社運営管理を標準化。

**エンティティ**: Asset, Equipment, Account, Posting

---

### M07 法務・契約・文書

**目的**: すべてのビジネス文書を一元管理。

**エンティティ**: Document, Contract, DocumentVersion

**機能**: アップロード、タグ、フォルダ、有効期限管理、バージョン管理、閲覧権限、更新タスク自動生成

---

### M08 経理・財務

**目的**: 経理実務と資金管理をバックオフィス全体と繋ぐ。

**エンティティ**: Transaction, Expense, Invoice, Payment, CashFlow

**サブモジュール**: 取引管理、経費精算、請求管理、支払管理、月次管理、CF管理、財務分析

---

### M09 報告・改善

**目的**: 問題発見と改善活動を日常業務に組み込む。

**エンティティ**: Report, ImprovementProposal

**フロー**: 投稿→対応者割当→改善タスク化→再発防止タスク化→経営者通知

---

### M10 経営管理

**目的**: 経営者の判断・改善のためのドリルダウン画面。M01からの遷移先。

---

### M11 ナレッジ・テンプレート

**目的**: 属人化防止、業務標準化。

**エンティティ**: Knowledge, Template

---

### M12 通知・期限管理

**目的**: 必要な情報を必要な人に必要なタイミングで届ける。

**通知チャネル**: アプリ内、メール、将来の外部連携用抽象化

**重複制御**: グルーピング、優先度、ミュート、ダイジェスト

---

### M13 組織・権限・監査

**目的**: マルチテナント、組織構造、権限、監査証跡の基盤。

**監査ログ対象**: 作成、編集、削除、承認、差戻し、アップロード、権限変更、ログイン、エクスポート
